# 数据结构设计

## 核心实体关系

```
教师 (Teacher)
  ├── 姓名、学科、学校
  └── 任教关系 → 班级（多对多，含时间范围）

班级 (Class)
  ├── 年级、班号、学年、状态（在教/已移交/已毕业）
  ├── 学生列表
  └── 任教教师（当前 + 历史）

学生 (Student)
  ├── 姓名、学号、入学年份（固定不变）
  ├── 当前年级（动态，随升学变化）
  └── 所属班级（可跨学年追踪）

课程 (Course)
  └── 按 年级 + 学期(上/下) 组织
       └── 单元 (Unit)
            └── 课时 (Lesson)
                 ├── 教案内容
                 ├── 教学目标
                 └── 知识点列表 (KnowledgePoint)

作业记录 (Assignment)
  ├── 关联：学生 + 课时
  ├── 批改结果、得分
  ├── 薄弱知识点
  └── 永久保留，跨学年可查
```

## 升学逻辑

1. 每学年结束，教师执行"升学操作"
2. 班级年级 +1（如三年级1班 → 四年级1班）
3. 学生当前年级 +1
4. 历史数据全部保留，新学年自动关联新年级课程
5. 六年级结束 → 班级状态变为"已毕业"

## 班级移交逻辑

1. 老师A执行"移交班级"→ 班级与老师A的任教关系结束
2. 老师B执行"接管班级"→ 班级与老师B建立新的任教关系
3. 老师A仍可查看自己任教期间的历史数据
4. 老师B可以看到班级的完整历史数据（含前任教师期间）
